# 人生の羅針盤 🧭

今の気持ちを書くと、それに合った名言が日本語で返ってくる小さなWebアプリです。

落ち込んだ夜やモヤモヤした朝に、そっと開いて気持ちを打ち込むと、
その気持ちに寄り添う名言が、日本語訳とAIからの一言つきで表示されます。

プログラミング初心者の自分が、「外部のAPIを自分のアプリにつなぐ」流れを
手を動かして理解するために作った課題作品です。

---

## どんなアプリ？

1. 「今の気持ち」を自由に文章で書く
2. 送信すると、AIが気持ちを読み取って、名言につける「タグ」を1つ選ぶ
3. そのタグで名言データベースから英語の名言を1つ取ってくる
4. 別のAIが、その英語の名言を自然な日本語に訳して、「なぜ今この言葉が響くか」の一言を添える
5. 名言（日本語訳）・原文・作者・AIの一言が画面に表示される

ポイントは、**AIを2回使っている**ところです。
1回目は「気持ち → タグ選び」、2回目は「英語の名言 → 日本語訳」。
1つ目の出力を2つ目の入力に渡す、という「APIをつなぐ」流れを体験するのがこの課題のテーマでした。

---

## データの流れ

```
[ユーザーが書いた気持ち]
        │
        ▼
[AI①：さくらのAI Engine] … 気持ちに合うタグを1つ選ぶ（例：failure）
        │
        ▼
[名言API：Quotable] … そのタグで英語の名言を取得
        │
        ▼
[AI②：さくらのAI Engine] … 英語を日本語に訳して、やさしい一言を添える
        │
        ▼
[画面に表示]
```

名言そのものはAIに作らせず、必ず名言APIから取っています。
（AIに名言を作らせると、存在しない言葉や作者を作ってしまうことがあるため）

---

## 使った技術

| 役割 | 使ったもの |
| --- | --- |
| 画面づくり | HTML / CSS / JavaScript（jQuery） |
| AI（タグ選び・翻訳） | さくらのAI Engine（モデル：gpt-oss-120b） |
| 名言の取得 | Quotable（のミラー版・APIキー不要） |

---

## 動かし方

このアプリはAPIキーを使うので、動かすには自分のキーを設定する必要があります。

1. `config.example.js` をコピーして、同じフォルダに `config.js` という名前で保存する
2. `config.js` の中の `"ここにあなたのAPIキーを貼る"` を、さくらのAI Engineで発行した本物のキーに書き換える
3. `index.html` をブラウザで開く

> ⚠️ `config.js`（本物のキーが入るファイル）は `.gitignore` で除外しているので、
> GitHubには上がりません。キーが外に漏れないようにするための仕組みです。

## 苦労した点 😵

### 1. 使う予定だったAPIが、いきなり使えなくなった
名言を取ってくる予定だった「Quotable」が、作っている途中で急に動かなくなりました。
ブラウザのエラーを見たら `ERR_CERT_DATE_INVALID`。調べると、**サーバー側の安全証明書の期限が切れていた**のが原因でした。
最初は「自分のコードが間違ってるのかな…」と何度も見直したのですが、原因は自分ではなくAPI側。
お店でいうと「営業許可証の期限が切れて閉店していた」ような状態でした。
最終的に、同じ仕組みを別の人が動かしている**ミラー版（姉妹サイト）に切り替えて**解決しました。
「予定していたものが使えないことは現場でも普通に起きる」「慌てず代わりを探して切り替える」というのを身をもって学べました。

### 2. ミラー版は「返事の形」も「タグの書き方」も違った
切り替えたミラー版は、本家とデータの返ってくる形が違っていました。
本家は `data[0].content` で名言が取れたのに、ミラー版は `data.quote.content`。
さらにタグも、本家は小文字 `wisdom` でよかったのに、ミラー版は**大文字始まりの `Wisdom`** でないと0件になる。
「ドキュメントを鵜呑みにせず、`console.log` で実物を見てから書く」ことの大事さを痛感しました。

### 3. 全角・半角の小さなミスでエラー
セミコロンをうっかり全角の `；` で打ってしまって動かない、`tags` の `s` を打ち忘れる…など、
ほんの1文字のミスで止まることが何度もありました。「記号は必ず半角」を体に叩き込みました。

## 頑張った点 💪

- **AIを2回つなぐ設計**をやりきったこと。「気持ち→タグ」「英語→日本語」と、1つ目の結果を2つ目に渡す流れを自分で組めたのが一番の達成感です。
- **APIキーを安全に管理する仕組み**を作ったこと。本物のキーは `config.js` に隔離して、`.gitignore` でGitHubに上がらないようにし、代わりに見本の `config.example.js` を用意しました。
- **エラーで白画面にならないようにした**こと。APIが落ちたときも、固まらずにエラーメッセージが出るようにしました（証明書切れのときも、これがちゃんと働いてくれました）。
- **AIの返事が崩れても大丈夫なように**したこと。AIがJSON以外を返してきても、`{` 〜 `}` だけを拾って読み取る「防御的な」処理を入れました。
- **AIの一言のトーンにこだわった**こと。説教くさくならず、読む人にそっと寄り添う「やさしい」口調になるようプロンプトを工夫しました。
- **コードに自分の言葉で日本語コメントを残した**こと。後から読み返しても、何をしているか分かるようにしました。

## 学んだこと

- URL・fetch・await・JSON といった、Webアプリの基本の仕組み
- APIには「鍵が要るもの（さくら）」と「要らないもの（Quotable）」があること
- ドキュメントの読み方（ベースURL＋パスの組み立て方）
- 「動かして確かめる」のが一番確実だということ（`console.log` で実物を見る）
- APIキーを外に漏らさないための管理のしかた

仕組みをもっとくわしくまとめた解説ページも `仕組み解説.html` に作りました。
